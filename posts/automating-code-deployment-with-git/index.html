Automating Code Deployment With Git

2017-12-01 13:24:34 2-minute read
Git
Git â€¢ hooks

Preface

After building an LNMP environment, you need to deploy the project code on the server, but using git to pull on the server side is too troublesome, so you need to make a git auto-deploy code feature, so that the project directory is automatically updated to the latest code after developers push on it. We need to use hooks.

Build a Git server

Installing Git 

1

sudo apt-get install git

Create a Git user to run the Git service 

1

sudo adduser git

Create certificate login

Put the developer’s public key, id_rsa.pub, into the /home/git/.ssh/authorized_keys file, one line at a time.

Initializing Git Repositories

Create a repository directory, note that it is a repository directory, not a project directory. Assuming /home/hub/app.git, enter the command in the /home/hub directory.

1

sudo git init --bare app.git

Git will then create a bare repository and give permissions to

1

sudo chown -R git:git app.git

Clone Remote Repository 

1

git clone git@yourserver:/home/hub/app.git

Add hooks for automatic deployment

Edit Script

Edit the /home/hub/app.git/hooks/post-receive file, or create a new one if you don’t have one. The contents are.

1

GIT_WORK_TREE=yourProjectDirectory   git checkout -f

Grant executable permissions

1

sudo chmod +x /home/hub/app.git/hooks/post-receive

Set the permissions for the project directory to the Git user

Since you are the Git user when you perform the pull, you need to set the permissions of the project directory to the Git user.

1

sudo chown -R git:git /var/www/html
