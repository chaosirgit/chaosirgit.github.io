Building LNMP From Scratch Tutorial

2017-10-08 13:08:09 3-minute read
LNMP
LNMP • Tutorial

Connect to the server via ssh

Server System Version Ubuntu 16.04

1

ssh root@127.0.0.1

Create a general account 

1

adduser username

Grant sudo privileges to the general account

Go to /etc/sudoers.d/ and create a new file with any name you want. Add username ALL=(ALL:ALL) ALL and you’re done.

Upgrade software source 

1

sudo apt-get update

Installing Mysql 

1

sudo apt-get install mysql-server mysql-client

mysql-server: MySQL server, the core program of mysql, is responsible for generating database instances and providing related interfaces for different clients to call. mysql-client: Is the client program, responsible for operating mysql instances. mysql-client is one of many client programs, other than mysql, mysqldump, mysqlslap, which are tools for accessing, backing up, and stress testing.

Test if MySQL is installed successfully 

1

mysql -uroot -p

Enter the password and the mysql> command line appears to indicate a successful installation.

Installing Nginx 

1

sudo apt-get install nginx

All configuration files are located in /etc/nginx/ after installation. (and each virtual host is already scheduled under /etc/nginx/sites-available).

  • The program files are in /user/sbin/nginx/
  • The log files are in /var/log/nginx
  • The startup scripts are in /etc/init.d/
  • The default project directory is in /var/www/

Starting Nginx 

1

sudo /etc/init.d/nginx start

or

1

sudo service nginx start

Test if nginx is installed successfully
Enter the URL and welcome nginx appears successfully.

Installing PHP 

1

sudo apt-get install php5-fpm php5-mysql

php5-fpm is a php FastCGI manager, originally a patch for php source code, due to its advanced nature, it was officially included after php5.3.3 and is no longer a third party package, now the downloadable php5-fpm is directly php source code and integrates branches of php5-fpm. php5-cgi is also a php FastCGI manager. It has many shortcomings, so don’t install php5-cgi after installing php5-fpm. php5-mysql is a mysql extension for php.

Open the nginx configuration file /etc/nginx/nginx.conf
Default virtual host configuration file /etc/nginx/sites-available/default

Installation.

Setting mysql default encoding set

View the default set of codes.

1

mysql> SHOW VARIABLES LIKE 'character%';

Add two parameters via the my.cnf file.

  1. Add in [mysqld]
1

default-character-set=utf8（ mysql 5.5 版本以上添加 character-set-server=utf8 ）
  1. Add in [client]
1

default-character-set=utf8

This way we don’t have to specify the utf8 character set when we build the database and tables. This way of writing in the configuration file solves the problem of data storage and comparison However, it does not work for client connections, which usually need to specify utf8 to avoid garbled code. This is also known as the set names command. In fact, the set names utf8 command corresponds to the following server-side commands.

1
2
3

SET character_set_client = utf8;
SET character_set_results = utf8;
SET character_set_connection = utf8;

However, these three parameters cannot be written in the configuration file my.cnf. They can only be changed dynamically with the set command. What we need is to write it in the configuration file once and for all Once and for all. So, is there a server-side solution to this problem, a possible idea is to set it in init_connect. This command is used in every can be set in the [mysqld] section by adding the following line to set the connection character set. Under [mysqld] add.

1

init_connect = 'SET NAMES utf8'
