Preface

Today’s project requires the business logic of querying the latest records based on grouping, and I want to use Eloquent to query out and make a record.

Raw SQL is: 

1

select * from (select * from project where user_id = :user_id order by id desc) as a group by a.name order by id desc;

Mysql 5.7 result as

idnameuser_idvalue
3Project 361test 3
2Project 261test 2
1Project 161test 1

The reason is that the 5.7 version of ORDER BY without LIMIT, less a DERIVED operation, it is estimated that the internal optimization, that ORDER BY in this syntax can be ignored, there is a LIMIT restriction involves sorting the results after the ORDER BY will not be ignored, you can achieve the expected.

Optimized to get 

1

select * from (select * from project where user_id = :user_id order by id desc limit 10000) as a group by a.name order by id desc;

<?php

public function projectList(Request $request){
        $limit = $request->get('limit',10);
        $user_id = $request->get('user_id',null);
        
        $sub_query = Project::where('user_id',$user_id)->orderBy('id','desc')->limit(1000);//Subquery
        
        $results = Project::select('*')
                ->from(DB::raw('('.$sub_query->toSql().') as a')) //from() is similar to DB::table(), toSql() gets the sql statement with ? sql statement
                ->mergeBindings($sub_query->getQuery())//mergeBindings() merges the binding parameters, getQuery() gets the specific values
                ->groupBy('name')
                ->orderBy('id','desc')
                ->paginate($limit);
        
        return $this->pageDate($results);
    }
?>
